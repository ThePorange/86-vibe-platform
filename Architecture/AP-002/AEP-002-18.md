# 1. Purpose

This document defines the Platform Reference Implementation for the 86-vibe platform.

The purpose of this specification is to consolidate the architectural decisions made throughout AP-001 and AP-002 into a single implementation blueprint. It establishes the recommended implementation sequence, maps architectural components to the physical repository structure, defines subsystem dependencies, and provides the authoritative hand-off from architecture to engineering.

Upon approval of this document, AP-002 SHALL be considered architecturally complete.

---

# 2. Scope

This specification defines:

- Reference implementation architecture
- Physical implementation mapping
- Component implementation order
- Package dependency relationships
- Repository implementation roadmap
- Engineering milestones
- Acceptance criteria
- Architecture completion criteria

This document does not redefine architecture previously established in AP-001 or AP-002.

---

# 3. Design Principles

The reference implementation SHALL adhere to the following principles.

## 3.1 Architecture Driven

Implementation SHALL follow approved architecture.

## 3.2 Incremental

Platform capabilities SHALL be implemented in manageable increments.

## 3.3 Testable

Each implementation increment SHALL be independently testable.

## 3.4 Traceable

Every implementation SHALL be traceable to one or more approved architecture documents.

## 3.5 Governed

Architecture SHALL remain the authoritative source for implementation.
