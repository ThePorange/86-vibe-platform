# 1. Purpose

This document defines the strategic roadmap for the development of the 86-vibe platform.

The roadmap describes the planned evolution of the platform through a series of capability-focused phases.

Implementation planning SHALL be managed through Jira and SHALL align with the roadmap defined in this document.

---

# 2. Roadmap Objectives

The roadmap SHALL:

- provide strategic direction
- define major platform capabilities
- establish implementation priorities
- communicate long-term objectives
- support incremental delivery
- minimise delivery risk
- enable continuous validation

The roadmap is intended to evolve through approved planning activities while remaining aligned with the product vision.

---

# 3. Product Vision Alignment

Every roadmap phase SHALL contribute directly to the vision established in AEP-001-02.

Each phase SHOULD:

- strengthen engineering governance
- improve AI-assisted software development
- increase platform capability
- improve developer productivity
- enhance engineering quality
- preserve architectural consistency

No roadmap item SHALL conflict with the Product Vision, Product Scope or Design Principles.

---

# 4. Release Roadmap

The initial roadmap for Version 1.x is organised into ten strategic phases.

| Phase | Name | Primary Outcome |
|------|------|-----------------|
| Phase 0 | Foundation | Establish platform architecture, governance and standards |
| Phase 1 | Core Platform | Build the core platform repository and documentation framework |
| Phase 2 | Documentation Engine | Deliver reusable documentation templates and document generation capabilities |
| Phase 3 | AI Agent Framework | Define and implement specialised engineering agents |
| Phase 4 | Knowledge Management | Introduce reusable engineering knowledge and project memory |
| Phase 5 | MCP Integration | Integrate external tools through the MCP layer |
| Phase 6 | OpenRouter Intelligence | Implement model routing, provider abstraction and cost optimisation |
| Phase 7 | Jira Automation | Automate engineering workflows through Jira |
| Phase 8 | Bootstrap Engine | Generate complete projects from standard templates |
| Phase 9 | Reference Application | Deliver a complete example project demonstrating the platform |

Successful completion of Phase 9 represents the planned Version 1.0 release of 86-vibe.

# 6. Success Criteria

Each roadmap phase SHALL satisfy the following criteria before completion.

- Planned deliverables completed.
- Documentation updated.
- Architecture remains compliant.
- Quality standards satisfied.
- Acceptance criteria achieved.
- Product Owner approval obtained.

Completion of a phase SHALL be formally recorded before subsequent phases begin.

---

# 8. Roadmap Governance

The roadmap SHALL be reviewed periodically.

Changes to the roadmap SHALL:

- support the Product Vision
- comply with Design Principles
- preserve architectural integrity
- be approved by the Product Owner

Implementation priorities MAY change without requiring modification of the roadmap, provided the strategic objectives remain unchanged.

---

# 9. Summary

The Product Roadmap establishes the strategic direction for the development of 86-vibe.

By organising development into capability-focused phases, the roadmap supports incremental delivery while maintaining alignment with the platform's architectural vision.

The roadmap provides the bridge between long-term product strategy and the detailed implementation planning managed through Jira.
