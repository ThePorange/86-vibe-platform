# 1. Purpose

This document defines the standard review process for Architecture Packages within the 86-vibe platform.

The objective is to ensure that architecture documents are complete, internally consistent, aligned with the Product Vision and suitable for implementation before approval.

This checklist SHALL be applied to all future Architecture Packages unless an approved Architecture Decision Record specifies an alternative process.

---

# 2. Review Objectives

Architecture reviews SHALL:

- verify architectural quality
- confirm alignment with product objectives
- identify inconsistencies
- verify traceability
- reduce implementation risk
- support long-term maintainability
- preserve engineering standards

The review process is intended to improve architecture before implementation begins.

---

# 3. Review Principles

The following principles apply to every architecture review.

## ARC-001 Completeness

All required documents SHALL be present.

---

## ARC-002 Consistency

Architecture documents SHALL be internally consistent.

---

## ARC-003 Traceability

Architecture SHALL remain traceable to product objectives and requirements.

---

## ARC-004 Review Before Implementation

Implementation SHALL NOT begin until the relevant Architecture Package has been approved.

---

## ARC-005 Evidence-Based Review

Review findings SHOULD be supported by documented evidence rather than personal preference.

---

# 4. Review Participants

The following roles participate in architecture reviews.

| Role | Responsibility |
|------|----------------|
| Product Owner | Confirms business alignment and approves the package |
| Chief Systems Architect | Reviews technical quality and architectural consistency |
| AI Review Agent (future) | Performs automated validation and identifies potential inconsistencies |
| Subject Matter Experts (optional) | Review specialised topics when required |

The Product Owner SHALL provide the final approval for an Architecture Package following completion of the review.

---

# 5. Architecture Review Checklist

Each item SHALL be reviewed before approval.

## Vision and Scope

- [ ] Product Vision remains supported.
- [ ] Product Scope remains unchanged or approved.
- [ ] Design Principles are respected.
- [ ] Governance Model is followed.

---

## Architecture

- [ ] Architectural decisions are documented.
- [ ] Components have clearly defined responsibilities.
- [ ] Interfaces are clearly described.
- [ ] Dependencies are understood.
- [ ] Architectural constraints are documented.
- [ ] No unnecessary complexity has been introduced.

---

## Traceability

- [ ] Requirements are traceable.
- [ ] Architecture references are complete.
- [ ] Relevant ADRs are referenced.
- [ ] Roadmap alignment is maintained.

---

## Documentation

- [ ] Documents are complete.
- [ ] Revision history is updated.
- [ ] Cross-references are correct.
- [ ] Diagrams are accurate.
- [ ] Terminology is consistent.

---

## Governance

- [ ] Required approvals have been obtained.
- [ ] Standards are followed.
- [ ] Risks have been reviewed.
- [ ] Success metrics remain applicable.

---

## Implementation Readiness

- [ ] Implementation scope is understood.
- [ ] Backlog alignment has been confirmed.
- [ ] Required dependencies have been identified.
- [ ] Acceptance criteria are defined where applicable.

---

# 6. Review Outcomes

Every Architecture Package SHALL conclude with one of the following outcomes.

| Outcome | Description |
|---------|-------------|
| Approved | Ready for implementation |
| Approved with Minor Actions | Implementation may proceed after completing agreed actions |
| Rework Required | Significant issues identified before approval |
| Rejected | Architecture does not satisfy review objectives |

Review outcomes SHALL be recorded within the Architecture Package review history.

---

# 7. Review Records

Architecture reviews SHOULD record:

- Review date
- Review participants
- Review outcome
- Outstanding actions
- Approval date
- Approved version

Review records SHALL be retained as part of the permanent architecture history.

---

# 8. Summary

The Architecture Review Checklist establishes a consistent and repeatable process for evaluating Architecture Packages within the 86-vibe platform.

By applying a structured review before implementation begins, the platform reduces architectural risk, improves engineering quality and preserves long-term maintainability.

All future Architecture Packages SHALL be reviewed using this checklist prior to implementation approval.
