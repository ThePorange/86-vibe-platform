# 1. Purpose

This document defines the mandatory completion criteria for engineering work undertaken within the 86-vibe platform.

The Definition of Done establishes a common understanding of when work is considered complete and ready for acceptance.

It applies to all engineering work regardless of whether it is performed by humans, AI agents or a combination of both.

---

# 2. Definition of Done Objectives

The Definition of Done SHALL:

- establish consistent completion criteria
- improve engineering quality
- reduce ambiguity
- support predictable delivery
- maintain documentation quality
- preserve traceability
- support release readiness

Work SHALL NOT be considered complete until the relevant Definition of Done has been satisfied.

---

# 3. Guiding Principles

The following principles apply to all engineering work.

## DOD-001 Completeness

Work SHALL satisfy all agreed acceptance criteria.

---

## DOD-002 Quality

Work SHALL comply with engineering standards.

---

## DOD-003 Documentation

Documentation SHALL be updated alongside implementation.

---

## DOD-004 Traceability

Engineering artefacts SHALL remain traceable to the approved Story, Epic and Architecture.

---

## DOD-005 Review

Work SHALL be reviewed before acceptance.

---

## DOD-006 Approval

Completed work SHALL be accepted by the Product Owner before closure.

---

# 4. Story Definition of Done

A Story is complete only when all of the following criteria have been satisfied.

## Planning

- [ ] Story approved.
- [ ] Acceptance criteria defined.
- [ ] Dependencies identified.

---

## Implementation

- [ ] Functionality implemented.
- [ ] Code follows engineering standards.
- [ ] No known critical defects remain.

---

## Testing

- [ ] Unit tests completed.
- [ ] Integration tests completed where applicable.
- [ ] Acceptance criteria validated.

---

## Documentation

- [ ] Documentation updated.
- [ ] Architecture references updated if required.
- [ ] Changelog updated where appropriate.

---

## Governance

- [ ] Code reviewed.
- [ ] Product Owner acceptance obtained.
- [ ] Story status updated in Jira.

---

# 5. Epic Definition of Done

An Epic is complete only when:

- [ ] All Stories are completed.
- [ ] All Stories satisfy the Story Definition of Done.
- [ ] Epic objectives achieved.
- [ ] Documentation complete.
- [ ] No unresolved critical defects remain.
- [ ] Product Owner approval obtained.

---

# 6. Architecture Package Definition of Done

An Architecture Package is complete only when:

- [ ] All planned documents completed.
- [ ] Internal consistency verified.
- [ ] Architecture Review Checklist completed.
- [ ] Required ADRs approved.
- [ ] Risks reviewed.
- [ ] Product Owner approval obtained.
- [ ] Baseline version recorded.
- [ ] Repository committed and tagged where appropriate.

---

# 7. Release Definition of Done

A Release is complete only when:

- [ ] Planned scope delivered.
- [ ] All required testing passed.
- [ ] Documentation published.
- [ ] Release notes completed.
- [ ] Changelog updated.
- [ ] Repository version updated.
- [ ] Git tag created.
- [ ] Product Owner approval obtained.
- [ ] Release artefacts published.

---

# 8. Evidence Requirements

Completion SHALL be supported by objective evidence.

Examples include:

- Git commits
- Pull Requests
- Test reports
- Architecture documents
- ADRs
- Jira Stories
- Release notes
- Review records

Evidence SHOULD be retained within version-controlled repositories wherever practical.

---

# 9. Exceptions

Exceptions to this Definition of Done SHALL:

- be documented
- identify the affected criteria
- record the rationale
- identify associated risks
- receive Product Owner approval

Approved exceptions SHALL remain traceable and SHALL be reviewed periodically.

---

# 10. Summary

The Definition of Done establishes the quality standard for engineering work performed within the 86-vibe platform.

By defining objective completion criteria for Stories, Epics, Architecture Packages and Releases, the platform ensures that engineering work is complete, traceable, documented and ready for long-term maintenance.

No engineering activity SHALL be considered complete until the applicable Definition of Done has been satisfied and approved.
