# 1. Executive Summary

86-vibe is an AI Engineering Platform that enables humans and AI agents to collaborate throughout the complete software development lifecycle using a structured, repeatable and governed engineering process.

Unlike AI coding assistants that focus primarily on generating source code, 86-vibe manages the entire lifecycle of software development—from initial idea through requirements, architecture, planning, implementation, testing, documentation, deployment and continuous improvement.

The platform is designed around one core principle:

> **Humans own product decisions. AI owns implementation.**

86-vibe provides the governance, documentation, workflows and automation required to ensure AI-assisted software development remains maintainable, auditable and scalable.

Rather than replacing software engineering practices, 86-vibe enhances them through intelligent automation while ensuring that humans remain responsible for strategic decisions.

---

# 2. Problem Statement

Recent advances in Large Language Models have dramatically increased the speed at which software can be produced.

Unfortunately, many AI-assisted projects suffer from common problems:

- undocumented decisions
- inconsistent architecture
- poor traceability
- uncontrolled technical debt
- lack of requirements
- duplicated functionality
- inconsistent testing
- changing coding styles
- limited project memory
- difficulty onboarding new contributors

Current AI coding tools generally optimise for code generation rather than software engineering.

As projects increase in size and complexity, these weaknesses become increasingly costly.

86-vibe addresses this problem by providing a structured engineering operating model that governs how AI participates within a software project.

---

# 3. Vision

To become the operating system for AI-assisted software engineering.

Every software project should be capable of being created, managed and maintained using a consistent engineering methodology regardless of programming language, cloud platform or AI provider.

86-vibe aims to make professional engineering practices accessible to individual developers, startups and enterprise engineering organisations alike.

---

# 4. Mission

The mission of 86-vibe is to enable humans and AI to build software together while maintaining the governance, quality and traceability expected of professional engineering teams.

The platform seeks to reduce development effort by automating repetitive engineering activities while preserving human ownership of product direction and architectural decisions.

---

# 5. Product Goals

The primary goals of 86-vibe are:

## Standardise Development

Provide a consistent software development methodology across every project.

---

## Improve Quality

Ensure every feature is fully documented, tested and traceable.

---

## Increase Automation

Automate repetitive engineering tasks including:

- documentation
- project setup
- backlog creation
- testing
- code review
- release preparation

---

## Preserve Engineering Knowledge

Capture architectural decisions, design rationale and project knowledge throughout the lifecycle of every project.

---

## Support Multiple AI Providers

Abstract AI providers behind a routing layer so that new models may be adopted without changing engineering processes.

---

## Remain Vendor Neutral

Avoid unnecessary dependencies upon any single:

- IDE
- cloud provider
- LLM
- project management tool
- source control provider

where practical.

---

# 6. Product Scope

Version 1.0 of 86-vibe focuses on providing an engineering platform rather than becoming an application development framework.

Core capabilities include:

- Product documentation
- Requirements management
- Architecture documentation
- AI agent orchestration
- Jira integration
- Git integration
- OpenRouter integration
- MCP integration
- Documentation generation
- Project bootstrap
- Quality assurance workflows
- Release management support

The platform intentionally excludes application-specific business functionality.

---

## Consultants

Professionals managing multiple client projects who require consistency and repeatability.

---

Small engineering teams seeking enterprise-quality engineering governance without enterprise overhead.

## Enterprise Engineering Teams

Communities wishing to coordinate AI-generated contributions while maintaining project quality.

# 8. Key Design Principles

The platform is founded upon the following principles.

## Human Governance

Humans retain ownership of:

- product vision
- prioritisation
- architecture approval
- release approval

AI assists but does not replace decision makers.

---

## Architecture First

No implementation begins until architecture has been approved.

---

## Documentation as a First-Class Deliverable

Documentation is produced alongside software rather than after implementation.

---

## Traceability

Every engineering artefact should be traceable from:

Vision

↓

Requirement

↓

Architecture

↓

Epic

↓

Story

↓

Implementation

↓

Test

↓

Release

---

## Modular AI

AI capabilities are divided into specialised agents with clearly defined responsibilities.

---

## Continuous Improvement

Every completed project contributes lessons learned back into the platform.

# 10. Success Criteria

The platform will be considered successful when it enables users to:

- create a new software project in minutes
- automatically generate complete engineering documentation
- maintain traceability throughout the project lifecycle
- coordinate multiple AI agents effectively
- reduce manual engineering effort
- improve software quality through automation
- support multiple AI providers without architectural changes

# Conclusion

86-vibe represents a shift from AI-assisted coding to AI-assisted software engineering.

Rather than treating AI as an autonomous software developer, the platform positions AI as a disciplined engineering team operating within a governed development framework.

By combining structured architecture, documentation, workflow automation, specialised AI agents and human oversight, 86-vibe aims to deliver software that is not only produced quickly but is also maintainable, auditable and scalable.

This document establishes the strategic foundation upon which all future architectural and implementation decisions for 86-vibe will be based.
